What is a regular expression?

Regular expressions is a sequence of characters that specifies a match pattern in text. Usually such patterns are used by string-searching algorithms for "find" or "find and replace" operations on strings, or for input validation.

Simple Regular Expressions

a regular expression literal in JS is written /pattern/ The simplest regexes simply match a given substring.

Character Classes

A "character class" matches only one out of several characters. To match an a or an e, use [ae]. You could use this in gr[ae]y to match either gray or grey.
In the example gray or grey matches the expression.üìù

Shorthand Character Classes

  • \d matches a single character that is a digit.
  • \w matches a "word character" (alphanumeric characters plus underscore).
  • \s matches a whitespace character (includes tabs and line breaks).
  • . matches any character but line breaks characters.
There are also shortend character classes.üìù

Anchors

Anchors do not match any characters. They match a position.

  • ^ matches at the start of the string.
  • $ matches at the end of the string.
  • \b matches at a word boundary.
A word boundary is a position between a character that can be matched by \w and a character that cannot be matched by \w. \b also matches at the start and/or end of the string if the first and/or last characters in the string are word characters.üìù

Repetition

  • ? makes the preceding token optional.
  • * the preceding token matches 0 or more times.
  • + the preceding token matches 1 or more times.

Regular Expression Flags

  • /g global; match/replace all occurrences.
  • /i case-insensitive.
  • /m multi-line mode.
  • /y The y (sticky) flag tells the regular expression to look for a match at lastIndex and only at lastIndex (not earlier or later in the string).
We will discuss particular examples of using g flag later.üìù

Use in JS - String.prototype.match()

If g flag is not used, work similar to exec function and returns first complete match and its related capturing groups.
If the g flag is used, all results matching the complete regular expression will be returned, but capturing groups are not included.
